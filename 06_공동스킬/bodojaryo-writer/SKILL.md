# 보도자료 작성 스킬 (강원특별자치도교육청 — 2026 신규 BI 서식)

## 개요

강원특별자치도교육청 문화체육특수교육과 장학사 업무 맥락에서
**언론 배포용 공식 보도자료**를 HWPX 파일로 작성하는 스킬이다.

- 양식: `assets/press-template.hwpx`
  — 2026 신규 BI **「모두가 빛나는 강원교육」** 공식 보도자료 서식
  — 상단: BI 시그니처(심벌+타이포) → 제호 표(보도자료/기관명, 배포일/과장 연락처)
    → 자료문의(하단 괘선) → 표제 상자(연노랑 배경 `#FFF3CD`) → 전문·본문 → 붙임
  — 모든 요소는 한글(HWP)에서 자유롭게 수정 가능(표·문단 배경·글자 모양 전부 편집 가능)
- 생성: `scripts/build_press.py` (press.json → HWPX 자동 생성)
- 현 교육감: **강삼영** — 전문·인용문에 반드시 반영

---

## ⚠️ 최우선 규칙 3가지

### 1. 과거형 원칙 (가장 중요!)

보도자료는 **행사가 끝난 후 기사화**되므로 서술은 **과거형으로 종결**한다.

| 구분 | 올바른 표현(과거형) | 금지 표현(예정형) |
|------|------------------|-----------------|
| 개최 | 개최했다, 열렸다 | 개최한다, 개최할 예정이다 |
| 운영 | 운영됐다, 진행됐다, 마련됐다 | 운영된다, 진행할 계획이다 |
| 시상 | 수여됐다, 시상했다 | 수여할 예정이다 |
| 발언 | 말했다, 밝혔다, 전했다, 강조했다 | — |

- 예외: 마지막 의의·전망 문단에서 **향후 계획**을 말할 때만
  "~해 나갈 계획이다", "~지원하겠다고 밝혔다" 허용
- 전문 첫 문장부터 과거형: "…'행사명'을 **개최했다**."
  ("오는 ○월 ○일 개최한다" ← 절대 금지)

### 2. 반드시 이 순서를 지킨다

```
① 입력 자료 확인 (운영계획·결과보고·수상자 명단 등)
       ↓
② 헤더 정보 확인 (배포일·과장·자료문의 담당자) — 없으면 사용자에게 질문
       ↓
③ 표제·부제 작성 (승부처! 아래 작성 기준 참조)
       ↓
④ 전문·본문 작성 (references/writing-guide.md 필독)
       ↓
⑤ press.json 작성 → scripts/build_press.py 실행
       ↓
⑥ 자가 품질 검토 2회 (과거형·숫자·고유명사 체크)
       ↓
⑦ HWPX 파일 출력 + 수정 필요 항목 안내
```

### 3. 사실 검증

- 날짜·장소·참가 규모·수상 내역 등 **숫자는 입력 자료와 대조** 후 기재
- 자료에 없는 사실은 지어내지 않는다 — 불명확하면 `○○` 처리 후 사용자에게 안내
- 교육감 인용문은 새로 작성하되, 행사 취지에 맞는 교육적 메시지로 구성

## 문서 구조 (서식 고정부 + 작성부)

| 구역 | 내용 | 처리 |
|------|------|------|
| BI 로고 | 심벌 + 모두가 빛나는 강원교육 | 양식 자동(수정 금지) |
| 제호 표 1행 | 보도자료 / 강원특별자치도교육청 | 양식 자동 |
| 제호 표 2행 | 배포일 / 과장 성명·전화 | `date`, `chief_line` |
| 자료문의 | ☎ · 담당 장학관 · 담당자 | `inquiry` |
| 표제 상자 | □ 표제 □ + "부제" (연노랑 배경) | `headline`, `subtitle` |
| 전문 | □ 육하원칙 리드 1문단 | `paragraphs` lv1 |
| 본문 | □ 문단 3~5개 + ◦ 세부 | `paragraphs` lv1/lv2 |
| 붙임 | 붙임 목록 + 끝. | `attachment` |

## 기호 체계

| 단계 | 기호 | 폰트 | 크기 | press.json 유형 |
|------|------|------|------|----------------|
| 표제 | □ ~ □ | HY헤드라인M 진하게 | 17pt | headline (기호 자동) |
| 부제 | " " | HY헤드라인M 진하게 | 13pt | subtitle (따옴표 자동) |
| 전문·본문 | □ | 휴먼명조 | 14pt | lv1 (기호 자동) |
| 세부 | ◦ | 휴먼명조 | 13pt | lv2 (기호 자동) |

> 기호는 스크립트가 자동으로 붙이므로 **텍스트에 □·◦를 포함하지 말 것**

## 헤더 기본값 (문화체육특수교육과)

사용자가 별도 지정하지 않으면 아래 기본값을 쓰되, **담당자 이름은 반드시 확인**한다.

- `chief_line`: `문화체육특수교육과장 : 이인범  ☎ 033) 258-5370`
- `inquiry`: `자료문의 : ☎ 258-5382  문화교육담당 장학관 ○○○  담당자 ○○○`
- 배포는 1일 1회 오전 10시 — 배포일 결정 시 참고(행사 종료 다음 날 이후 권장)

```bash

  "inquiry": "자료문의 : ☎ 258-5382  문화교육담당 장학관 ○○○  담당자 ○○○",
  "headline": "2026 강원학생 하모니 페스티벌 성황리 개최",
  "subtitle": "35개교 928명, 나흘간 예술로 하나 된 강원교육",
  "paragraphs": [

    ["lv1", "이번 페스티벌에는 도내 35개교 학생 928명이 참가해 ..."],
    ["lv2", "아카펠라·중등합창(7. 13.~14. 소공연장), 오케스트라·초등합창(7. 15.~16. 신사임당홀)"],
    ["lv1", "강삼영 교육감은 \"...\"라고 말했다."],
    ["lv1", "강원특별자치도교육청은 앞으로도 ... 운영해 나갈 계획이다."]
```

- 인용문 내부 큰따옴표는 JSON 이스케이프(`\"`)로 넣는다.

## 자가 품질 검토 체크리스트 (2회 실시)

1. 모든 서술이 과거형으로 끝나는가? (예정형 잔존 여부 전수 확인)
2. 전문에 누가·언제·어디서·무엇을이 모두 담겼는가?
3. 교육감 성명이 **강삼영**으로 정확한가?
4. 날짜·요일·장소·인원·학교명이 입력 자료와 일치하는가?
5. 표제가 행사명+핵심 성과를 담아 한 줄로 명쾌한가?
6. 문단 수 3~5개 내외, 각 문단 1~3문장으로 간결한가?
7. 붙임 표기가 "붙임  ○○ ○매.  끝." 형식인가?
8. ObjectFinder로 치환 결과를 확인했는가?

## 주의사항

1. **양식 우선**: 사용자가 다른 보도자료 양식(.hwpx)을 업로드하면 그 양식을 사용
   (이때는 hwpx 스킬의 ZIP-level 치환 워크플로우 적용)
2. 생성 후 `ObjectFinder`로 텍스트 전수 확인 (hwpx 스킬 참조)
3. 레이아웃 최종 확인은 한글 앱에서 — 페이지 나눔은 한글이 결정
4. 표제·부제가 길면 자동 줄바꿈됨(정상). 2줄 초과 시 문구 축약 권고
5. 보도자료는 1쪽 분량이 기본, 최대 2쪽을 넘기지 않는다
