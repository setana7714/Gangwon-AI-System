# HWP/HWPX 문서 처리 스킬

## 목적

교육청 업무용 한글 문서(HWP/HWPX)를 읽고, 구조를 분석하고, 수정본 또는 새 문서를 HWPX 형식으로 만든다. 공문, 계획서, 보고서, 보도자료 등 교육청 문서의 제목, 본문, 표, 붙임, 글머리표 구조를 최대한 유지한다.

원본 HWP/HWPX 문서 파일은 스킬 폴더에 저장하지 않는다. 샘플 문서를 참고할 때는 문서 구조와 처리 규칙만 추출하고, 문서번호·학교명·개인정보·민감정보는 원본 그대로 저장하지 않는다.

## 지원 기능

1. HWP 파일 내용 읽기
2. HWPX 파일 내용 읽기
3. HWP 문서를 HWPX 문서로 변환
4. 기존 HWPX 문서 수정
5. HWPX 새 문서 생성
6. 제목, 본문, 표, 붙임, 글머리표 구조 유지
7. 공문·계획서·보고서·보도자료 문서 구조 분석
8. 확인되지 않은 정보는 `[확인 필요]`로 표시
9. 출력은 기본적으로 HWPX 형식으로 생성
10. 원본 문서는 절대 직접 수정하지 않고 복사본 또는 새 파일로 생성
11. 파일 생성 후 열림 여부와 문서 무결성 확인

## 기본 workflow

1. 입력 파일 형식이 HWP인지 HWPX인지 확인한다.
2. 원본 파일을 직접 수정하지 않고 작업용 복사본 또는 새 출력 경로를 정한다.
3. 문서에서 제목, 본문, 표, 붙임, 글머리표, 문단 흐름을 추출한다.
4. 공문·계획서·보고서·보도자료 중 어떤 구조인지 분류한다.
5. 요청에 따라 요약, 변환, 수정, 새 문서 생성을 수행한다.
6. 확인되지 않은 정보는 `[확인 필요]`로 표시한다.
7. 결과 파일은 기본적으로 HWPX로 생성한다.
8. 생성 후 파일이 열리는지, 필수 XML과 패키지 구조가 정상인지 확인한다.

상세 형식과 점검 기준은 `references/format-guide.md`를 읽고 따른다.

## 작업 유형별 지침

### HWP 파일 내용 읽기

- 가능한 도구로 텍스트를 추출한다.
- 표와 글머리표가 있으면 일반 문단과 분리해서 표시한다.
- 추출이 불완전하면 `[추출 확인 필요]`로 표시한다.
- 원본 HWP 파일을 스킬 폴더나 GitHub 관리 대상에 복사하지 않는다.

### HWPX 파일 내용 읽기

- HWPX는 ZIP 패키지로 열어 XML 구조를 확인한다.
- 본문 XML에서 문단, 표, 글머리표, 붙임 목록을 추출한다.
- 단순 문자열 치환 전에 문단 구조와 표 구조를 확인한다.

### HWP를 HWPX로 변환

- 원본 HWP는 그대로 보존한다.
- 출력 파일명은 원본명 뒤에 `_변환.hwpx` 또는 사용자가 지정한 이름을 사용한다.
- 변환 후 열림 여부와 텍스트 누락 여부를 확인한다.

### 기존 HWPX 문서 수정

- 원본 HWPX를 직접 수정하지 않는다.
- 수정본은 `_수정본.hwpx`, `_작성본.hwpx`, `_완성본.hwpx` 등 새 이름으로 저장한다.
- 제목, 본문, 표, 붙임, 글머리표 구조를 유지한다.
- 변경 대상이 불명확하면 원문을 보존하고 `[수정 확인 필요]`로 표시한다.

### HWPX 새 문서 생성

- 출력은 기본적으로 HWPX로 생성한다.
- 문서 유형에 따라 공문, 계획서, 보고서, 보도자료 구조를 선택한다.
- 서식 파일이 없으면 본문 중심의 편집 가능한 HWPX 초안을 만든다.
- 사용자가 별도 양식을 제공하면 그 양식을 우선한다.

## 문서 구조 분석 기준

- 공문: 수신, 경유, 제목, 근거, 요청·알림 내용, 붙임, 끝 표시
- 계획서: 표지, 추진 근거, 목적, 추진 개요, 세부 내용, 예산, 기대 효과, 행정사항
- 보고서: 개요, 추진 결과, 주요 내용, 문제점, 향후 계획, 붙임
- 보도자료: 배포 정보, 자료문의, 표제, 부제, 전문, 본문, 붙임

문서 유형이 섞여 있으면 사용자의 최종 목적을 우선한다.

## 정보 보호 규칙

- 실제 문서번호, 학교명, 개인명, 전화번호, 이메일, 계좌번호를 스킬 파일에 저장하지 않는다.
- 학생, 교직원, 민원, 감사, 징계, 건강, 장애, 가정환경 등 민감정보는 원문 그대로 남기지 않는다.
- 예시가 필요하면 `[문서번호 확인 필요]`, `[학교명 확인 필요]`, `[담당자 확인 필요]`처럼 비식별 표현을 사용한다.

## 무결성 확인

HWPX 생성 또는 수정 후 다음을 확인한다.

- HWPX 파일이 ZIP 패키지로 열리는가
- `mimetype`과 `Contents/` 등 기본 구조가 유지되는가
- 본문 XML이 깨지지 않았는가
- 표 병합, 셀 수, 글머리표 구조가 손상되지 않았는가
- 다시 열었을 때 텍스트가 누락되지 않았는가

문제가 있으면 결과를 납품하지 말고 새 파일로 다시 생성한다.
