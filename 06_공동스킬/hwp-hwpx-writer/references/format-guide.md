# HWP/HWPX 처리 형식 안내

## 1. 기본 원칙

한글 문서 작업은 원본 보존을 최우선으로 한다.

- 원본 파일을 직접 수정하지 않는다.
- 수정본 또는 변환본은 새 파일명으로 저장한다.
- 출력 기본 형식은 HWPX로 한다.
- 확인되지 않은 정보는 `[확인 필요]`로 표시한다.
- 원본 문서 파일과 민감정보는 스킬 폴더에 저장하지 않는다.

## 2. 파일명 규칙

권장 출력 파일명:

```text
[원본파일명]_변환.hwpx
[원본파일명]_수정본.hwpx
[원본파일명]_작성본.hwpx
[문서제목]_초안.hwpx
[문서제목]_완성본.hwpx
```

파일명에 개인정보나 민감정보가 있으면 일반화한다.

## 3. HWP 읽기

HWP는 바이너리 형식이므로 환경에서 가능한 도구를 사용해 텍스트를 추출한다.

읽기 결과는 다음처럼 정리한다.

```text
문서 제목: [제목 확인 필요]
문서 유형: 공문/계획서/보고서/보도자료/[유형 확인 필요]

주요 내용:
1. ...
2. ...

표:
- [표 제목 또는 위치]
- 주요 열: ...

붙임:
- ...

확인 필요:
- [추출 확인 필요]
```

텍스트 추출이 불완전하면 추정하지 말고 `[추출 확인 필요]`로 표시한다.

## 4. HWPX 읽기

HWPX는 ZIP 기반 패키지로 보고 처리한다.

확인 대상:

- `mimetype`
- `Contents/`
- 본문 XML
- 헤더와 스타일 XML
- 이미지와 첨부 리소스

문서 내용을 읽을 때는 문단 순서를 유지하고, 표는 행과 열 구조를 보존해 추출한다.

## 5. HWPX 수정

수정 절차:

1. 원본 파일 복사본을 만든다.
2. 수정 대상 문구 또는 위치를 확인한다.
3. 문단, 표, 붙임 중 어느 영역인지 구분한다.
4. 구조가 단순하면 해당 텍스트만 교체한다.
5. 표나 병합 셀이 있으면 셀 구조를 먼저 확인한다.
6. 수정 후 HWPX 패키지를 다시 열어 무결성을 확인한다.

불명확한 변경은 다음처럼 표시한다.

```text
[수정 확인 필요]
[문구 확인 필요]
[위치 확인 필요]
```

## 6. HWPX 생성

새 문서를 만들 때는 문서 유형별 기본 구조를 선택한다.

### 공문

```text
수신  [수신자 확인 필요]
제목  [제목 확인 필요]

1. 관련: [문서번호 확인 필요]

2. [본문 내용]

붙임  [붙임명 확인 필요] 1부.  끝.
```

### 계획서

```text
[제목] 운영 계획

Ⅰ. 근 거
Ⅱ. 목 적
Ⅲ. 추진 개요
Ⅳ. 세부 추진 내용
Ⅴ. 예산
Ⅵ. 기대 효과
Ⅶ. 행정사항
```

### 보고서

```text
[제목] 결과 보고

1. 개요
2. 추진 결과
3. 주요 내용
4. 문제점 및 개선 방안
5. 향후 계획
붙임
```

### 보도자료

```text
보도자료
자료문의: [담당자 확인 필요]

[표제]
[부제]

□ [전문]
□ [본문]
◦ [세부 내용]

붙임  [붙임명 확인 필요] 1부.  끝.
```

## 7. 표 처리

표가 포함된 문서는 다음을 유지한다.

- 열 개수
- 행 개수
- 병합 셀
- 헤더 행
- 합계 행
- 금액 쉼표
- 비고 열

표 구조가 불명확하면 표를 임의로 재구성하지 말고 `[표 구조 확인 필요]`로 표시한다.

예산표는 합계가 맞는지 확인하고, 맞지 않으면 `[합계 확인 필요]`로 표시한다.

## 8. 글머리표와 붙임

문서의 기존 글머리표 체계를 유지한다.

자주 쓰는 체계:

```text
1. 2. 3.
가. 나. 다.
1) 2) 3)
가) 나) 다)
□
○
-
※
```

붙임은 문서 끝에 둔다.

붙임 1개:

```text
붙임  [붙임명 확인 필요] 1부.  끝.
```

붙임 여러 개:

```text
붙임  1. [붙임명 확인 필요] 1부.
      2. [붙임명 확인 필요] 1부.  끝.
```

## 9. 문서 구조 분석

문서를 분석할 때 다음 항목을 추출한다.

```text
문서 유형:
문서 제목:
작성 목적:
핵심 대상:
주요 날짜:
예산 또는 금액:
표 포함 여부:
붙임 목록:
확인 필요:
```

문서 유형 판단 기준:

- 공문: 수신, 제목, 관련, 붙임, 끝
- 계획서: 목적, 추진 개요, 세부 추진 내용, 예산, 기대 효과
- 보고서: 추진 결과, 실적, 문제점, 향후 계획
- 보도자료: 배포일, 자료문의, 표제, 부제, 전문, 본문

## 10. 무결성 확인

HWPX 파일 생성 후 다음을 확인한다.

- ZIP으로 열리는가
- 필수 파일과 폴더가 존재하는가
- XML 파싱 오류가 없는가
- 본문 텍스트가 비어 있지 않은가
- 표 구조가 깨지지 않았는가
- 다시 열었을 때 오류 메시지가 없는가

확인 결과 문제가 있으면 납품 전에 새 파일로 다시 만든다.

## 11. 비식별 처리

샘플 문서를 참고할 때 저장 가능한 것은 구조와 규칙뿐이다.

저장 금지:

- 원본 HWP/HWPX 파일
- 실제 문서번호
- 실제 학교명
- 개인명, 연락처, 이메일
- 계좌번호
- 학생·교직원 관련 개인정보
- 감사, 민원, 징계, 건강, 장애 등 민감정보

비식별 표현:

```text
[문서번호 확인 필요]
[학교명 확인 필요]
[담당자 확인 필요]
[연락처 확인 필요]
[민감정보 비식별 처리]
```
