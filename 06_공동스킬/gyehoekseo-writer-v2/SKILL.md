# 계획서 작성 스킬 v2 (강원특별자치도교육청 — 2026 신규 BI 파랑 양식)

> **v2 변경 요약 (2026. 7.)**
> - ① 섹션 번호 로마자(Ⅰ~Ⅶ) ↔ 아라비아(1~6) 선택 가능 — `numeral` 값만 바꾸면 됨
> - ② 표 삽입 워크플로우 신설 (`scripts/insert_tables.py`) — 일정·예산·점검표 등
> - ③ **표 병합 버그 수정**: 구판은 병합 시 '덮인 셀'이 남아 행 colSpan 합이 열 수를 초과 →
>   **한글이 표를 '손상'으로 인식해 편집 불가·파일 삭제**를 유발했음. v2는 덮인 셀을 제거하고
>   마스터에 span을 부여해 각 행 colSpan 합 = colCnt를 보장(생성 시 자동 그리드 검증).
> - ④ 표지 긴 제목 2줄·문단 테두리(선) 증식 방지 주의 명문화

## 개요

강원특별자치도교육청 문화체육특수교육과 장학사 업무 맥락에서
**사업·행사·연수 운영계획서**를 HWPX 파일로 작성하는 스킬이다.

- 양식: `assets/report-template.hwpx`
  — 2026 신규 BI **「모두가 빛나는 강원교육」** 기준 (네이비 `#005078` + BI 블루 `#1BA8E1`)
  — 표지: 심볼(좌상단) + 로고타입(우상단), 제목 위아래 네이비 이중선, 하단 엠블럼·기관명
  — 본문: 슬로건 → 제목(위아래 이중선 내장) → 부서명 → 섹션 Ⅰ~Ⅶ
  — 제목의 이중선은 **제목 문단 자체의 테두리** — 한글에서 [문단 모양 → 테두리/배경]으로
    선 종류·굵기·색·간격을 자유 조정 가능 (빈 문단 방식이 아니라 실수로 지워지지 않음)
- 생성: `scripts/build_plan.py` (plan.json → HWPX 자동 생성)
- 구 양식(더나은 강원교육·주황 톤)은 폐기됨. 반드시 이 스킬의 assets 양식만 사용.

## ⚠️ 최우선 규칙: 반드시 이 순서를 지킨다

```
① 입력 자료 확인 (사업계획 내용 / 보고서 / 관련 공문)
       ↓
② 계획서 섹션 구조 결정 (Ⅰ~Ⅶ 중 필요한 섹션 선택)
       ↓
③ 각 섹션 내용 작성 (아래 섹션 기준 참조)
       ↓
④ plan.json 작성 → scripts/build_plan.py 실행
       ↓
⑤ 자가 품질 검토 2회 (체크리스트)
       ↓
⑥ HWPX 파일 출력 + 수정 필요 항목 안내
```

> 사용자가 내용만 주고 형식을 지정하지 않으면 → 표준 섹션 구조로 작성

## 1. 표준 섹션 구조

### 표지 (1쪽) — 양식 자동, 제목·날짜만 치환

- 제목: 사업명 + "운영 계획" (예: `2026 악기뱅크 2.0 울림 프로젝트 운영 계획`)
- 날짜: `2026. ○.` 형식 (월까지만 표기)
- 긴 제목(공백 포함 18자 이상, 예: 26자 이상)은 표지에서 자동 2줄 처리됨 — 정상 동작.
  build_plan.py는 제목 텍스트만 치환하므로 위·아래 이중선은 문단 전체를 감싸(위선=첫 줄 위,
  아래선=끝 줄 아래) 글자를 관통하지 않는다.
- 심볼·로고타입·엠블럼·기관명은 양식에 포함 — 수정 금지
- ⚠️ **한글에서 표지 편집 시 주의(선 증식 방지)**: 제목 이중선은 **제목 문단의 문단 테두리**다.
  선이 있는 문단 안에서 Enter를 치면 같은 테두리 문단이 복제되어 선이 늘어난다.
  → 위치를 내리려면 **선 없는 빈 줄에서 Enter**, 이미 늘어난 선은 그 빈 문단에 커서 두고
  Backspace로 문단째 삭제, 또는 [문단 모양 → 테두리/배경]에서 테두리 '없음' 처리.

### 본문 섹션 구성 (2쪽~)

| 섹션 | 제목 | 필수 여부 | 내용 |
|------|------|----------|------|
| Ⅰ | 근 거 | 필수 | 관련 법령·지침·공문 근거 |
| Ⅱ | 목 적 | 필수 | 사업 추진 목적 (3줄 이내) |
| Ⅲ | 방 향(방침) | 권장 | 운영 기본 방향·원칙 |
| Ⅳ | 추진 개요 | 필수 | 대상·기간·예산 등 핵심 개요 |
| Ⅴ | 세부 추진 내용 | 필수 | 과제별·단계별 세부 내용 |
| Ⅵ | 기대 효과 | 권장 | 사업 기대 효과 |
| Ⅶ | 행정사항 | 권장 | 예산·보안·보고 등 행정 처리사항 |

> 섹션 수는 사업 규모에 따라 조정 가능. 최소 Ⅰ·Ⅱ·Ⅳ·Ⅴ는 포함.
> 생략 시에도 로마자 번호는 Ⅰ부터 **순서대로 재부여하지 않고 원래 번호 유지**
> (예: Ⅲ 생략 시 Ⅰ, Ⅱ, Ⅳ, Ⅴ… 순으로 표기)

#### 섹션 번호 표기: 로마자(기본) ↔ 아라비아 숫자 선택 가능

`plan.json`의 `numeral` 값은 스크립트가 뱃지에 **그대로** 출력한다.
따라서 문서 성격에 맞게 두 방식 중 하나를 쓴다(스크립트 수정 불필요).

| 방식 | numeral 값 | 적합한 문서 |
|------|-----------|------------|
| 로마자(기본) | `"Ⅰ" "Ⅱ" "Ⅲ" … "Ⅶ"` | 사업·행사·연수 운영계획서 등 정식 계획서 |
| 아라비아 | `"1" "2" "3" … "6"` | 모니터링·협의회 계획, 간이 운영계획 등 원본이 1.·2. 체계인 경우 |

- 아라비아 예시(모니터링 계획): `1 목적 / 2 방침 / 3 세부 운영 계획 / 4 예산 / 5 기대 효과 / 6 행정사항`
- 아라비아 방식일 때 `근 거` 섹션이 없으면 `1 목적`부터 시작해도 무방(원본 관례 우선).

## 2. 기호 체계 (신규 양식 기준 — 공문과 다름!)

| 단계 | 기호 | 폰트 | 크기 | 굵기 | 문단 위 | plan.json 유형 |
|------|------|------|------|------|--------|---------------|
| 섹션 헤더 | `Ⅰ Ⅱ Ⅲ …` | HY헤드라인M (네이비 뱃지+제목) | 16pt | 진하게 | 자동 | `numeral`/`title` |
| 1단계 | `□` | HY헤드라인M | 14pt | 진하게 | 15 | `lv1` |
| 체크항목 | `✔` (블루) | 휴먼명조 | 13pt | 보통 | 10 | `check` |
| 2단계 | `○` | 휴먼명조 | 13pt | 보통 | 10 | `lv2` |
| 3단계 | `―` | 휴먼명조 | 13pt | 보통 | 6 | `lv3` |
| 4단계 | `※` | 휴먼명조 | 13pt | 보통 | 3 | `lv4` |

- 서체 체계(2026. 7. 확정): 로마자 섹션 **헤드라인M 16** / □ **헤드라인M 14** /
  나머지 전부 **휴먼명조 13 보통 굵기**
- 정렬: 본문 전체 왼쪽 정렬 (양식에 내장)
- ⚠️ 공문의 `1. → 가. → 1) → 가)` 체계와 완전히 다름. 혼용 금지.

## 3. 섹션별 작성 기준

### Ⅰ 근 거 — `check`
법령·지침·공문 순으로 나열:
```
✔ 2026년 강원교육 주요업무계획(문화예술교육 활성화)
✔ 강원특별자치도교육청 문화체육특수교육과-○○○○(2026. ○. ○.)
```

### Ⅱ 목 적 — `check`
간결한 완성형 문장, 3항목 이내.

### Ⅲ 방 향(방침) — `check`
운영 기본 원칙 4항목 이내.

### Ⅳ 추진 개요 — `lv1`~`lv4`
반드시 포함: 사업명 / 대상(학교·학생 수) / 기간 / 장소(해당 시) / 예산 / 추진 주체

### Ⅴ 세부 추진 내용 — `lv1`~`lv4`
과제 구분이 있을 경우:
```
□ 과제 1: [과제명]
  ○ 대상 / 운영방법
    ― 세부내용
      ※ 참고사항
```

### Ⅵ 기대 효과 — `check`
3항목 이내, 간결한 완성형 문장.

### Ⅶ 행정사항 — `lv2`·`lv3`
예산 집행 기준 / 결과보고 기한·제출처 / 보안(개인정보) 등.

## 4. HWPX 생성 절차

```bash
# ① plan.json 작성 (아래 구조)
# ② 실행
python <스킬경로>/scripts/build_plan.py plan.json /mnt/user-data/outputs/계획서.hwpx
```

plan.json 구조:
```json
{
  "title": "2026 ○○사업 운영 계획",
  "date": "2026. 7.",
  "sections": [
    {"numeral": "Ⅰ", "title": "근 거",
     "items": [["check", "근거 내용"]]},
    {"numeral": "Ⅳ", "title": "추진 개요",
     "items": [["lv1", "사업명: ○○"], ["lv2", "대상: ○○"],
               ["lv3", "세부"], ["lv4", "참고"]]}
  ]
}
```

주의:
- item 텍스트에 기호(✔□○―※)를 포함하지 말 것 — 스크립트가 자동 부착
- 섹션 제목은 양식 관례대로 띄어쓰기: `근 거`, `목 적`, `방 향(방침)`, `기대 효과`
- 첫 섹션은 자동으로 본문 상단 간격(14pt), 이후 섹션은 22pt 적용
- 스크립트가 mimetype 우선 재패키징까지 수행 — 별도 네임스페이스 후처리 불필요

### 표 삽입 (일정표·예산표·점검표 등) — `scripts/insert_tables.py`

`build_plan.py`는 텍스트 문단만 생성한다. 표가 필요하면 **자리표시(placeholder)
→ 표 교체** 2단계로 처리한다(python-hwpx 사용, 표 테두리·병합·헤더 음영 지원).

```
① plan.json 항목에 자리표시를 lv2로 넣는다:  ["lv2", "@@TBL_SCHEDULE@@"]
② python build_plan.py plan.json 텍스트본.hwpx
③ tables.json 작성
④ python insert_tables.py 텍스트본.hwpx tables.json 최종.hwpx
```

tables.json 구조(자리표시 이름 → 표 정의):
```json
{
  "@@TBL_SCHEDULE@@": {
    "widths": [6, 7, 7, 11, 15, 9, 30, 12],
    "header": ["연번", "학교급", "지역", "학교명", "일시", "장소", "운영주제", "담당자"],
    "rows":   [["1", "초", "강릉", "사천초", "9. 8.(화) 15:00~", "인근 카페", "…", "이소민"]],
    "shade_header": true
  },
  "@@TBL_BUDGET@@": {
    "widths": [16, 62, 18],
    "header": ["사업항목", "산출내역", "지출금액"],
    "rows":   [["간담회비", "…산출근거…", "270,000"], ["합    계", "", "270,000"]],
    "merges": [[2, 0, 2, 1]],
    "shade_header": true
  }
}
```
- `widths`: 열 너비 가중치(열 수와 동일)
- `merges`: `[시작행,시작열,끝행,끝열]`(끝 좌표 포함). v2는 병합 시 **덮인 셀을 제거**하고 마스터에
  span을 부여 → 각 행 colSpan 합 = colCnt 보장. 생성 직후 **그리드 정합을 자동 검증**하며,
  불일치 시 경고를 출력한다(구판의 '표 손상→편집 불가' 재발 방지).
- 표 셀 글자는 자리표시 문단의 charPr(휴먼명조 13pt)을 승계 — 별도 지정 불필요
- 설치 필요: `pip install python-hwpx --break-system-packages` (세션마다 확인)
- 표 삽입 뒤에도 mimetype 우선·STORED 재패키징은 python-hwpx가 자동 처리

> 붙임(첨부) 문단·표를 본문 뒤에 이어 붙일 때도 같은 방식으로 `section.add_paragraph(
> text, para_pr_id_ref=…, char_pr_id_ref=…)` + `paragraph.add_table(...)`를 쓴다.
> 스타일 id는 본문 기호 문단에서 수집(□=53/21, ○=54/22, ―=55/22, ※=60/22 기준, 양식 버전에 따라 상이하니 실측).

### 그 밖의 미세 수정 (스크립트 범위 밖)

특정 문단 스타일 변경 등은 **hwpx 스킬**(`/mnt/skills/user/hwpx/SKILL.md`)의
ZIP-level XML 편집 절차를 따른다. 이때 양식 무결성 검증:
```python
import zipfile, re
s = zipfile.ZipFile(OUT).read('Contents/section0.xml').decode('utf-8')
h = zipfile.ZipFile(OUT).read('Contents/header.xml').decode('utf-8')
used = set(re.findall(r'paraPrIDRef="(\d+)"', s))
defined = set(re.findall(r'<hh:paraPr\b[^>]*?(?<![A-Za-z])id="(\d+)"', h))
assert not (used - defined)
```

## 5. 자가 품질 검토 체크리스트 (2회 수행)

- [ ] 표지: 제목·날짜·발행기관이 정확한가
- [ ] 본문 제목이 표지 제목과 일치하는가
- [ ] 섹션 구성: Ⅰ·Ⅱ·Ⅳ·Ⅴ가 모두 포함되었는가
- [ ] 로마자 번호가 순서대로인가 (Ⅰ→Ⅶ)
- [ ] 기호 체계: `□ → ○ → ― → ※` 위계가 일관되는가
- [ ] 근거: 관련 법령·공문 문서번호가 정확한가
- [ ] 추진 개요: 대상·기간·예산 수치가 명시되었는가
- [ ] 예산: 아라비아 숫자 + 한글 병기 형식인가 (`금377,000,000원(금3억7천7백만원)`)
- [ ] 결과보고 기한이 행정사항에 명시되었는가
- [ ] 플레이스홀더(양식 샘플 텍스트: "휴먼명조 13포인트" 등)가 남아 있지 않은가
- [ ] HwpxDocument.open()으로 재개방 검증을 했는가
