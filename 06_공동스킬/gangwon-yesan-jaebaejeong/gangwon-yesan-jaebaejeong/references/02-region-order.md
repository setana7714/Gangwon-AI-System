# 02. 지역청 정렬 순서 · 표준명 변환

## 지정 순서 (절대 준수)
```
춘천 → 원주 → 강릉 → 속초양양 → 동해 → 태백 → 삼척 →
홍천 → 횡성 → 영월 → 평창 → 정선 → 철원 → 화천 →
양구 → 인제 → 고성
```
- 가나다순·엑셀 자동정렬이 아니라 **반드시 위 지정 순서**로 정렬한다.
- 사용자가 제시한 지역청만 위 순서대로 정리한다. 자료에 없는 지역청은 제외한다.
- 전체 지역청 틀이 필요한 경우에만 17개를 모두 표시하고, 해당 없는 지역은 빈칸 또는 0으로 처리한다.
- 코드에서는 `reallocation_lib.REGION_ORDER`, `order_regions()` 사용.

## 표준명 변환 (원자료 → 표준)
| 원자료 표기 | 표준 |
|---|---|
| 춘천시, 춘천교육지원청 | 춘천 |
| 원주시, 원주교육지원청 | 원주 |
| 강릉시, 강릉교육지원청 | 강릉 |
| 속초, 양양, 속초·양양, 속양 | 속초양양 |
| 동해시 | 동해 |
| 태백시 | 태백 |
| 삼척시 | 삼척 |
| 홍천군 | 홍천 |
| 횡성군 | 횡성 |
| 영월군 | 영월 |
| 평창군 | 평창 |
| 정선군 | 정선 |
| 철원군 | 철원 |
| 화천군 | 화천 |
| 양구군 | 양구 |
| 인제군 | 인제 |
| 고성군 | 고성 |

- 지역명 표기는 위 표준명으로 통일한다. 코드: `reallocation_lib.normalize_region()`.
- 매핑에 없는 표기가 나오면 임의 변환하지 말고 원문을 유지한 뒤 검토표에 "지역청 표기 확인 필요"로 남긴다.
