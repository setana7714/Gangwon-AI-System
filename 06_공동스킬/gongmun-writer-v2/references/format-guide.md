# 공문 형식 안내

## 1. 공통 구조

공문 초안은 아래 구조를 기본으로 한다.

```text
수신  [수신자]
(경유) [경유자, 없으면 생략]
제목  [공문 제목]

1. [근거 또는 배경]

2. [핵심 안내 또는 요청 내용]

3. [기한, 제출 방법, 협조 사항]

붙임  [붙임명] 1부.  끝.
```

수신자, 문서번호, 학교명, 개인명, 전화번호, 금액, 제출 기한 등이 확인되지 않으면 임의로 채우지 말고 `[확인 필요]`로 표시한다.

## 2. 문체 규칙

- 공공기관 문체를 사용한다.
- 문장은 짧고 명확하게 쓴다.
- 요청은 `~하여 주시기 바랍니다`로 정중하게 쓴다.
- 알림은 `~을 다음과 같이 알려드리니 참고하여 주시기 바랍니다`를 기본으로 한다.
- 제출 요청은 `기한 내 제출하여 주시기 바랍니다`를 기본으로 한다.
- 통보는 `~ 결과를 다음과 같이 알려드립니다` 또는 `~ 알려드리니 후속 조치하여 주시기 바랍니다`를 사용한다.
- 재배정은 `목적에 맞게 집행하여 주시기 바랍니다`와 `집행 및 정산에 철저를 기하여 주시기 바랍니다`를 사용한다.

## 3. 본문 작성 기준

### 1항: 근거 또는 배경

관련 계획, 공문, 사업 추진 배경을 쓴다.

예시:

```text
1. 관련: [문서번호 확인 필요]
```

근거가 없거나 불명확하면 생략하지 말고 `[문서번호 확인 필요]`로 둔다.

### 2항: 핵심 내용

알림, 요청, 선정 결과, 재배정 등 공문의 주된 내용을 쓴다.

예시:

```text
2. 2026년 [사업명 확인 필요] 운영을 다음과 같이 알려드리니, 대상 학교에서는 업무 추진에 참고하여 주시기 바랍니다.
```

### 3항: 기한과 방법

제출 기한, 제출처, 담당자 연락처, 유의 사항을 쓴다.

예시:

```text
3. 아울러 관련 자료를 [제출 기한 확인 필요]까지 [제출 방법 확인 필요]로 제출하여 주시기 바랍니다.
```

## 4. 유형별 문안

### 내부결재용

```text
제목  [사업명] 추진 계획(안) 보고

1. 관련: [문서번호 확인 필요]

2. [사업명]을 다음과 같이 추진하고자 하오니 검토하여 주시기 바랍니다.

붙임  [계획서명] 1부.  끝.
```

### 외부발송용

```text
수신  [수신자 확인 필요]
제목  [사업명] 안내

1. 관련: [문서번호 확인 필요]

2. [사업명]을 다음과 같이 안내하오니, 해당 업무가 원활히 추진될 수 있도록 협조하여 주시기 바랍니다.

붙임  [붙임 파일명 확인 필요] 1부.  끝.
```

### 알림 공문

```text
제목  [행사명 또는 사업명] 알림

1. 관련: [문서번호 확인 필요]

2. [행사명 또는 사업명]을 다음과 같이 알려드리니, 대상자는 참석 또는 업무 추진에 참고하여 주시기 바랍니다.
```

### 제출 요청 공문

```text
제목  [자료명] 제출 요청

1. 관련: [문서번호 확인 필요]

2. [자료명] 제출을 다음과 같이 요청하오니, 기한 내 제출하여 주시기 바랍니다.

  가. 제출자료: [자료명 확인 필요]
  나. 제출기한: [제출 기한 확인 필요]
  다. 제출방법: [제출 방법 확인 필요]
```

### 선정 결과 통보 공문

```text
제목  [사업명] 선정 결과 알림

1. 관련: [문서번호 확인 필요]

2. [사업명] 선정 결과를 다음과 같이 알려드리니, 선정 기관에서는 후속 절차를 추진하여 주시기 바랍니다.

붙임  선정 결과 1부.  끝.
```

### 재배정 공문

```text
제목  [사업명] 사업비 재배정 알림

1. 관련: [문서번호 확인 필요]

2. [사업명] 운영을 위한 사업비를 다음과 같이 재배정하오니, 목적에 맞게 집행하여 주시기 바랍니다.

  가. 재배정 대상: [대상 확인 필요]
  나. 재배정 금액: [금액 확인 필요]
  다. 집행 기간: [기간 확인 필요]
  라. 정산 및 결과보고: [기한 확인 필요]

3. 사업비 집행 시 관련 지침을 준수하고, 집행 및 정산에 철저를 기하여 주시기 바랍니다.

붙임  사업비 재배정 내역 1부.  끝.
```

## 5. 붙임 목록 규칙

붙임이 1개일 때:

```text
붙임  [붙임명] 1부.  끝.
```

붙임이 여러 개일 때:

```text
붙임  1. [붙임명] 1부.
      2. [붙임명] 1부.  끝.
```

붙임명에 실제 학생명, 교직원명, 학교별 민감 자료명이 포함되면 일반화한다.

## 6. 비식별 처리 규칙

스킬 안에는 다음 정보를 원본 그대로 저장하지 않는다.

- 실제 문서번호
- 실제 학교명 또는 개인명
- 전화번호, 이메일, 계좌번호
- 학생, 교직원, 민원 관련 개인정보
- 내부 검토 의견, 감사, 징계, 민원 등 민감정보

필요한 경우 다음처럼 바꿔 쓴다.

```text
[문서번호 확인 필요]
[학교명 확인 필요]
[담당자 확인 필요]
[연락처 확인 필요]
[민감정보 비식별 처리]
```
